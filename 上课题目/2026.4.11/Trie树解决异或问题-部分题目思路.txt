Trie树解决异或问题

有一个数组A，长度n，每个0 <= A[i] <= 1e6
在数组A中任选两个数字A[i]和A[j]
目标：A[i] ^ A[j] 最大，问这个最大值是几？

暴力：O(n^2)的枚举

Trie树的做法：把每个数字作为二进制串，插入Trie树中
	固定长度：当A[i] <= 1e6，可以认为这些数字都是20位的二进制数
	不足20位的，在高位补0
	按照从高位到低位的顺序，插入Trie树
	这个Trie树是二叉树 
	
	对于每个A[i]，在Trie树上找和它异或最大的结果
	从高到低位：如果Trie树中可以走和A[i]这一位不同的分支，则往下走
		否则，只能走另一条路
		
P4551：求从根到节点i的异或和，作为A[i]
		这样从i到j的路径异或和，就变成了A[i] ^ A[j]
		（i和j的lca到根的这一段异或和 被 异或抵消） 

Trie树实现平衡树set/multiset
	把每个数字都看作二进制串，保证长度一致
	只有长度一致的情况下，数字的大小 与 字典序的大小 就是一致的

	插入：按照Trie树的插入方法插入即可
		数组v[i]表示以i为根的子树中有几个数字
		在插入时，路过的节点v[i]++

	删除：如果数字不存在，什么都不做
		如果存在：需要把路过的点v[i]--
		事先把路过的点放到数组中

		如果是直接把一个键出现的所有次数都删掉
		 	则v[i] -= x，其中x表示键出现的次数
 	
	Trie树中往下走：节点存在，并且v[i] != 0 
	 
	multiset<int> S;
	只删某个键出现的一次，S.erase(S.find(x))
	把一个键所有出现次数都删掉，S.erase(x)

	找x是集合中的第几个数字				ord 	(order，顺序)
	或：找集合中有多少个数字 <= x 
		 在Trie树上沿着x的二进制走
		 如果走0，序号不变
		 如果走1，序号加上左子树的大小
 
	集合中第k小的数字是几？ 			rnk 	（rank，序号） 
		判断左子树的大小v[i]和k的关系
		如果v[i]>=k，就继续搜索左子树
		否则，搜索右子树，序号变成k-v[i]

	合并两个集合：递归合并


P6824
	a[i]都插入Trie树中，按20位二进制串插入
	统计子树大小

	枚举x，在Trie树中比较x ^ a[i]和k从高到低位的二进制

	当计算到第i位：
		如果k的第i位是0，那么只能沿着异或后结果是0的分支向下走
		如果k的第i位是1，那么答案加上异或后结果是0的分支大小
					然后沿着异或后结果是1的分支向下走

P11209
	对于离线T = 0，查询的答案和上次的答案没关系
	对于在线T = 1，查询的答案要和上次的答案做异或

	对于一组ai和bi，从高到低位依次考虑
	对于第j位
	如果ai和bi的第j位都是0，k的第j位必须是0
	如果ai和bi的第j位都是1，k的第j位可以是0，也可以是1
		 如果k的第j位取0，需要继续向下走
		 如果k的第j位取1，则后面任何二进制组合都满足条件
	如果ai第j位是0，bi的第j位是1
		如果k的第j位取1，需要继续向下走
		如果k的第j位取0，则后面任何二进制组合都满足条件
	如果ai第j位是1，bi的第j位是0，k的第j位必须是1

	bi的第j位是0，k的第j位必须和ai一样，需要继续搜索
	bi的第j位是1，k的第j位如果和ai一样，后面怎样组合都可以
				 k的第j位如果和ai不一样，需要继续搜索
		 
	插入Trie树的就是对k进行搜索的过程

	const int B = 30;
	void insert(int i)
	{
		int r = 0;
		for(int j = B - 1; j >= 0; j--)
		{
			if((b[i] >> j) & 1)
			{
				int bit = (a[i] >> j) & 1;
				int& nr = nxt[r][bit];
				if(!nr) nr = ++tid;
				v[nr]++;
				int& xr = nxt[r][1 - bit];
				if(!xr) xr = ++tid;
				r = xr;
			}
			else
			{
				int& nr = nxt[r][(a[i] >> j) & 1]; 
				if(!nr) nr = ++tid;
				r = nr;
			}
		}
		v[r]++;
	}
